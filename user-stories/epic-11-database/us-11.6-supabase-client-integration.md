# US-11.6 — Supabase client integration

**As a developer**, I want a typed Supabase client set up in the codebase, so that all database operations use a consistent, type-safe interface.

## Acceptance criteria

- `@supabase/supabase-js` is installed as a dependency
- `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` are documented in `.env.example`
- `src/lib/supabase.ts` exports an initialised, typed Supabase client
- TypeScript types reflect the database schema (`src/lib/database.types.ts`)
- Client returns `null` when env vars are missing; app falls back to localStorage (offline mode)
