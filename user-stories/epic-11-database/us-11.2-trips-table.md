# US-11.2 — Trips table

**As a developer**, I want a `trips` table that stores all trip metadata, so that each user's trips are persisted to the database.

## Acceptance criteria

- `trips` table created via Supabase migration with columns:

| Column | Type | Notes |
|---|---|---|
| `id` | `uuid` | Primary key |
| `user_id` | `uuid` | Foreign key → `users.id` |
| `title` | `text` | Trip name |
| `destination` | `text` | |
| `start_date` | `date` | |
| `end_date` | `date` | |
| `adults` | `int` | Min 1 |
| `children` | `int` | Min 0 |
| `pets` | `int` | Min 0 |
| `pet_type` | `text` | e.g. "dog" |
| `budget` | `numeric` | Total budget in € |
| `holiday_type` | `text` | beach / mountains / city / camping / other |
| `pet_friendly` | `boolean` | Default false |
| `created_at` | `timestamptz` | Default: `now()` |

- RLS enabled: users can only read/write their own trips (`user_id = auth.uid()`)
- Index on `user_id` for fast list queries
